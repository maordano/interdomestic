Interacciones Bióticas

Guía conceptual, protocolo de búsqueda, y generador de consulta - v1.0

Nota¿Qué son las interacciones bióticas?

Una interacción biótica es cualquier tipo de interacción entre formas de vida orgánicas a cualquier nivel de organización biológica.

A nivel de personas o individuos biológicamente identificados, se define como una relación entre seres vivos, sean personas humanas o no, y se clasifican de varias maneras, como muchas cosas del entorno doméstico.

Hay muchos tipos de interacciones bióticas. Una relación mínima como la comunicación es una interacción, hay al menos una persona receptora, hay al menos una persona emisora, y la información va y viene.

Definir el estado del conocimiento sobre temas complejos es un camino meticuloso, que parece más fácil a medida que se acerca la singularidad.

Las interacciones bióticas son abordadas típicamente desde la ecología y la evolución, sin embargo, está siendo abordada por muchas ramas disciplinares de la construcción del conocimiento, de manera determinada o indirectamente. Sumado a ello, desde los sistemas de clasificación de la vida, las interacciones bióticas nos incluyen.

Las interacciones bióticas son la pieza conceptual sobre la que se construye interdomestic: caracterizar cómo se relacionan las especies en el ámbito doméstico requiere primero y clasificar las diversas especies y formas de la vida silvestre con las que nos relacionamos.

Como un ejemplo tipológico, en este diagrama se representan las principales clases de interacciones entre especies.

Por figurar un ejemplo de búsqueda de conocimiento aplicado a partir de múltiples interacciones bióticas; en la agricultura, en el manejo ambiental, y en la eco-epidemiología; las interacciones también se clasifican con criterio funcional y aplicable a la relación humana-naturaleza: los servicios ecosistémicos, o servicios de la naturaleza a la vida humana; y sus contrapartes, los dis-servicios ecosistémicos, con procesos también derivados de interacciones bióticas que perjudican la vida humana.

En esta aplicación hacemos el ejercicio de elegir 2 especies sobre las cuales interesa buscar información para una revisión sistematizada.

El armado de búsqueda se puede filtrar por palabras clave en torno a las interacciones bióticas en el ámbito doméstico, y generar un documento de consulta para usar como prompt en buscadores de literatura de primera fuente, o en su asistente de IA de preferencia (modelo híbrido de construcción del conocimiento).

NotaProtocolo de búsqueda — PRISMA aplicado

Para buscar literatura sobre una interacción específica, un paso clave de la construcción del conocimiento -además del resultado-, es conocer cómo fue generado el conocimiento, o sea, documentar el proceso, — es lo que exige un protocolo tipo PRISMA, y lo que ya armamos en detalle en la ficha de Meta-análisis de este mismo apéndice (los 12 campos de “Search strategy” de la GSRRF de OSF).

Los cuatro pasos clásicos de PRISMA, aplicados a un ejemplo de interacción:

  1. Identificación — se define el par de especies y los conceptos de interés (por ejemplo, Psychotria carthagenensis y Turdus rufiventris, con foco en dispersión de semillas), y se arma la consulta de búsqueda.

  2. Cribado — se revisan los títulos/resúmenes de lo que la búsqueda trae, descartando lo que no menciona a ninguna de las dos especies o al fenómeno de interés.

  3. Elegibilidad — de lo que queda, se lee el texto completo de cada documento candidato para confirmar que documenta evidencia “real” -de fuente primaria en la base de conocimiento- de la interacción (por ejemplo, que descarte una mención al paso sobre una de las especies, mientras que se elige el que documenta datos procesables de la interacción).

  4. Inclusión — las referencias que pasan los tres filtros anteriores forman el “conjunto final”, documentado con su cita completa y — cuando la interacción esté formalmente registrada — su ficha en GloBI, la base de datos de interacciones bióticas documentadas (ver también Apps en este apéndice).

Los pasos 2 a 4 son trabajo profundo de lectura, estudio y criterio; procesos cuyos contenidos van más allá de los alcances de esta herramienta.

Esta herramienta ayuda con el paso 1: armar una consulta o prompt estructurada y documentada, lista para llevar al buscador (o al asistente de IA) de su preferencia.

NotaGenerador de consulta

Elige un par de especies y, opcionalmente, los conceptos que te interesa enfocar.

La app arma una consulta de búsqueda prolija y un prompt listo para pegar en un asistente de IA, sin ejecutar ninguna búsqueda por sí misma. Es liviana (esperar un minuto en la primera carga): solo valida los nombres científicos con GBIF y el resto es texto.

Puede descargar la ficha de consulta generada como archivo .md, legible en dispositivos mediados por humanos y legible por máquinas; así como directamente derivar a plataformas de búsqueda de fuente primaria de conocimiento.

# Reutilizado de bio-distribution.qmd: valida nombres contra el backbone de GBIF
gbif_name_match <- function(name) {
  url <- paste0("https://api.gbif.org/v1/species/match?name=", URLencode(name))
  tryCatch(fromJSON(url), error = function(e) NULL)
}

items_planos <- do.call(c, unname(lapply(grupos_keywords, function(g) g$items)))

generar_checkboxes_grupo <- function(grupo_id, grupo) {
  choices <- setNames(names(grupo$items), sapply(grupo$items, function(x) x$label))
  wellPanel(
    style = paste0("background: ", grupo$color_fondo, "; border: 1px solid ", grupo$color_borde, "; margin-bottom: 8px;"),
    tags$strong(style = "font-size: 0.85em;", grupo$titulo),
    tags$p(style = "font-size: 0.78em; color: #64748b; margin: 2px 0 6px 0; font-style: italic;", grupo$descripcion),
    checkboxGroupInput(paste0("cb_", grupo_id), NULL, choices = choices, inline = TRUE)
  )
}

# ==========================================
# GENERADORES: arman texto, no ejecutan ninguna búsqueda
# ==========================================
generar_consulta <- function(especie_a, especie_b, rasgos_ids) {
  base <- paste0('"', especie_a, '" "', especie_b, '"')
  if (length(rasgos_ids) == 0) return(base)
  keywords <- sapply(items_planos[rasgos_ids], function(x) x$keyword)
  paste(base, paste(keywords, collapse = " OR "))
}

generar_prompt_llm <- function(especie_a, especie_b, rasgos_labels) {
  foco <- if (length(rasgos_labels) > 0) paste0(", con foco en: ", paste(rasgos_labels, collapse = ", ")) else ""
  paste0(
    "Necesito referencias bibliográficas sobre la interacción biótica entre ", especie_a, " y ", especie_b, foco, ". ",
    "Prioriza fuentes que documenten evidencia directa, fuentes primarias de base de conocimiento y publicaciones sobre esta interacción entre estas dos especies puntuales, ",
    "además de fuentes de la historia natural y ecología de cada especie a partir de búsquedas separadas para cada especie. Para cada referencia indica: autor(es), año, título, ",
    "y DOI o enlace si está disponible. Si no encuentras evidencia directa de la interacción entre estas dos ",
    "especies, explicita en vez de aproximar o inventar una referencia."
  )
}

# ==========================================
# SERVIDOR
# ==========================================
server <- function(input, output, session) {

  validacion <- eventReactive(input$verificar, {
    list(
      a = list(consulta = input$especie_a, match = gbif_name_match(input$especie_a)),
      b = list(consulta = input$especie_b, match = gbif_name_match(input$especie_b))
    )
  })

  output$validacion_nombres <- renderUI({
    v <- validacion()
    tagList(lapply(list(v$a, v$b), function(r) {
      m <- r$match
      if (is.null(m) || is.null(m$matchType) || m$matchType == "NONE") {
        div(class = "alert alert-danger", style = "padding:6px; font-size:0.85em; margin-top:4px;",
            paste0("❌ '", r$consulta, "' no se encontró en el backbone de GBIF."))
      } else if (m$matchType == "EXACT") {
        div(class = "alert alert-success", style = "padding:6px; font-size:0.85em; margin-top:4px;",
            paste0("✅ '", r$consulta, "' reconocido como ", m$scientificName, " (", m$status, ")"))
      } else {
        div(class = "alert alert-warning", style = "padding:6px; font-size:0.85em; margin-top:4px;",
            paste0("⚠️ '", r$consulta, "' es una coincidencia aproximada. GBIF sugiere: ", m$canonicalName, "."))
      }
    }))
  })

  rasgos_elegidos_r <- reactive({
    c(input$cb_sector, input$cb_eco_evolutiva, input$cb_historia_vida, input$cb_aplicado)
  })

  resultado <- eventReactive(input$generar, {
    req(input$especie_a, input$especie_b)
    ids <- rasgos_elegidos_r()
    labels <- if (length(ids) > 0) {
      etiquetas <- sapply(items_planos[ids], function(x) if (is.null(x)) NA_character_ else x$label)
      etiquetas[!is.na(etiquetas)]
    } else {
      character(0)
    }
    list(
      consulta = generar_consulta(input$especie_a, input$especie_b, ids),
      prompt = generar_prompt_llm(input$especie_a, input$especie_b, labels),
      rasgos = labels
    )
  })

  output$resultado_panel <- renderUI({
    res <- resultado()
    req(res)
    scholar_url <- paste0("https://scholar.google.com/scholar?q=", URLencode(res$consulta))
    crossref_url <- paste0("https://search.crossref.org/?q=", URLencode(res$consulta), "&from_ui=yes")

    wellPanel(
      style = "background: #ffffff; border: 1px solid #d1d5db;",
      tags$h4("Consulta generada"),
      tags$p(style = "font-size: 0.8em; color: #64748b;",
             if (length(res$rasgos) > 0) paste0("Enfocada en: ", paste(res$rasgos, collapse = ", ")) else "Sin enfoque adicional (solo las dos especies)."),
      if (length(res$rasgos) > 1) div(class = "alert alert-warning", style = "padding:6px; font-size:0.78em;",
          "⚠️ Marcar varios conceptos los combina con OR, lo cual ensancha la búsqueda en vez de precisarla — por eso a veces aparecen resultados de muchas disciplinas y fuentes primarias o secundarias. Para mayor precisión, prueba con un solo concepto por búsqueda."),
      verbatimTextOutput("consulta_texto"),
      tags$div(
        style = "display: flex; gap: 8px; flex-wrap: wrap; margin: 8px 0;",
        tags$a(href = scholar_url, target = "_blank", class = "btn btn-sm btn-primary", "Buscar en Google Scholar"),
        tags$a(href = crossref_url, target = "_blank", class = "btn btn-sm btn-outline-secondary", "Buscar en Crossref"),
        tags$a(href = "https://www.globalbioticinteractions.org", target = "_blank", class = "btn btn-sm btn-outline-secondary", "Buscar en GloBI")
      ),
      tags$hr(),
      tags$h5("Prompt sugerido para un asistente de IA"),
      verbatimTextOutput("prompt_texto"),
      br(),
      downloadButton("descargar_registro", "Descargar registro (.md)", class = "btn-outline-secondary")
    )
  })

  output$consulta_texto <- renderText({ resultado()$consulta })
  output$prompt_texto <- renderText({ resultado()$prompt })

  output$descargar_registro <- downloadHandler(
    filename = function() paste0("interdomestic_consulta_", Sys.Date(), ".md"),
    content = function(file) {
      res <- resultado()
      req(res)

      especie_a <- if (is.null(input$especie_a) || !nzchar(input$especie_a)) "(sin especificar)" else input$especie_a
      especie_b <- if (is.null(input$especie_b) || !nzchar(input$especie_b)) "(sin especificar)" else input$especie_b
      rasgos_txt <- if (length(res$rasgos) > 0) paste0("- ", res$rasgos, collapse = "\n") else "Ninguno"
      consulta_txt <- if (is.null(res$consulta) || !nzchar(res$consulta)) "(no disponible)" else res$consulta
      prompt_txt <- if (is.null(res$prompt) || !nzchar(res$prompt)) "(no disponible)" else res$prompt
      fecha_txt <- tryCatch(format(Sys.time(), "%Y-%m-%d %H:%M"), error = function(e) as.character(Sys.time()))

      texto <- paste0(
        "# Registro de consulta — interdomestic\n\n",
        "Generado: ", fecha_txt, "\n\n",
        "## Especies\n\n- Especie A: ", especie_a, "\n- Especie B: ", especie_b, "\n\n",
        "## Conceptos de enfoque\n\n", rasgos_txt, "\n\n",
        "## Consulta para buscador\n\n", consulta_txt, "\n\n",
        "## Prompt sugerido para asistente de IA\n\n", prompt_txt, "\n"
      )
      cat(texto, file = file)
    }
  )
}

shinyApp(ui, server)
